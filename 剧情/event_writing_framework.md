# 职场苏丹 - 事件编剧框架

## 导言

本文档为《职场苏丹》游戏编剧提供一个结构化的创作框架，帮助你设计符合游戏机制的高质量事件内容。事件系统采用"卡牌槽位决定分支"的设计理念，要求编剧在创作时同时考虑叙事内容和游戏机制。

## 事件创作工作流程

1. **事件概念构思**：确定事件的核心概念和目标
2. **事件结构设计**：设计卡牌槽位和分支结构
3. **叙事内容创作**：为每个分支编写具体内容
4. **机制数值设计**：设置属性条件和结果影响
5. **事件链接规划**：设计与其他事件的联系
6. **多语言本地化**：准备中英文内容
7. **平衡性检查**：确保事件设计平衡
8. **技术实现准备**：准备JSON格式数据

## 编剧最佳实践

1. **保持选择有意义**：每个槽位和选项应代表明显不同的策略方向
2. **平衡风险与回报**：高风险选项应有更高回报，低风险选项回报适中
3. **多样化结果**：不同选择应导向足够差异化的结果
4. **关注属性范围**：设计条件时考虑玩家在该阶段可能的属性范围
5. **事件链思维**：思考事件如何融入更大的故事线
6. **角色一致性**：确保NPC反应符合其性格特点
7. **提供有用信息**：在描述中为玩家提供选择所需的信息
8. **槽位描述明确**：让玩家理解每个槽位代表什么策略方向
9. **成功失败清晰**：明确传达什么条件下会成功或失败
10. **中英文表达一致**：确保两种语言表达相同的意思和情感

## 示例：办公室会议事件

```
事件ID：10001
事件名称（中文）：紧急部门会议
事件类型：日常事件
事件标签：职场,会议,挑战
有效回合：3-10
事件优先级：50
是否可重复：否
最大触发次数：1

场景描述（中文）：部门经理突然召集了一个紧急会议。各部门负责人都在场，似乎有重要事项要讨论。作为新任经理，你需要为自己部门的表现负责。

槽位1：
槽位ID：presentation_slot
槽位描述（中文）：选择一种方式来展示你部门的工作
是否必填：是
允许的卡牌类型：presentation,leadership,technical
效果加成：
- 社交：x 1.2
- 智力表达：x 1.5
关联分支ID：confident_presentation
成功条件：
- 魅力：≥ 70
- 准备：≥ 60
失败条件：
- 魅力：≤ 30
成功结果：
- 属性变化：声望 +15，压力 +5
- 关系变化：与老板关系 +10，与同事张关系 -5
失败结果：
- 属性变化：声望 -10，压力 +15
- 关系变化：与老板关系 -5
成功后触发事件：boss_praise_001
失败后触发事件：null

分支：confident_presentation
分支文本：
- 中文段落1：你自信地展示了团队的工作和成果，数据清晰，逻辑严密。部门经理看起来很满意，其他与会者也频频点头。

角色反应：
- 角色1 [boss]：
  - 情绪：pleased
  - 对话（中文）：做得很好！这正是我们需要的那种清晰的思路。
- 角色2 [colleague_zhang]：
  - 情绪：jealous
  - 对话（中文）：（小声嘀咕）看来有人准备充分啊...

场景变化：
- 视觉效果：会议室的气氛变得活跃
- 音效：轻微的掌声.mp3
```
